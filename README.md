## 功能介绍
### 1. 画面信号发生器
配合 CalMAN / ColourSpace 等支持无线发生器的校准软件使用（就类似dogegen ）。支持 SDR / P3 广色域 / HDR 三种模式，应该是目前安卓发生器中独一份。
- 无线 TCP 连接（端口 20002）
- 支持 APL 窗口大小调整
- SDR / P3 / HDR 模式切换
> 需要网络条件良好，穿墙或强干扰环境可能不稳定。
个人体验验证是没问题的，HDR部分与HDR10 PQ体验视频对照没有区别。

### 2. 拖影体验
文字流从下向上滚动模拟阅读场景，三组灰阶文字（G255 / G200 / G128），关注两点：
- **运动清晰度**：是否能在滚动中看清文字
- **RGB Motion Breaking**：观察文字是否有RGB分离（紫色拖影、文字绿边），代表 R/G/B 像素响应不一致
建议选择 G17/G38 作为背景以模拟真实 APP 场景。

### 3. APL体验
生成 100→10 APL 的画面，测量不同 APL 下的亮度变化。
- 100-80 APL 是日常实际使用区间，重点关注
- 50 APL 以下改为 1:1 正方形，方便采样

### 4. 均匀度体验
全屏纯色灰阶检测屏幕均匀性。基于大量 APP 采样选定灰阶：
| 灰阶 | 用途 |
|------|------|
| G255 | 全屏白——基础检测 |
| G38 | 深色模式常用低灰阶（大部分较深色APP） |
| G17 | 深色模式常用低灰阶（对应WX） |
| G10 | 斗蛐蛐专区（真没用，好能表演一下差也是正常的） |

### 5. Ultra HDR 灰阶体验
21 点 Ultra HDR EOTF 灰阶，Base 2.2。纯看个激发图一乐吧。

### 6. 次像素渲染体验（SPR）
1 像素硬线条图案，配合显微镜观察 OLED 次像素排列。
- **如果你有find X3啥的最方便，二手可以收一个（bushi）

### 7. HDR 倍率扫描
读取屏幕SDR码值和当前HDR/SDR 倍率，也就是显示HDR时，SDR部分亮度低了N倍：
- 自动控制亮度 2%→100%，20 步
- 生成完整折线图（横轴 SDR 码值，纵轴 HDR 倍率）
- 支持保存图表到相册
这个测试应该核心关注低码值区间，我真的被半夜爆闪搞怕了

### 8. RGB 体验（杂项全扔这里了）
#### a.广色域对比
左半 Display P3、右半 sRGB，检测手机是否支持广色域声明。

#### b.RGB Gain Map
全屏 R/G/B 纯色条，左半增益 4X/4X/4X，右半 R=1X / G=2X / B=3X。体验手机是否支持三通道增益图。长按切换到 SDR 对比。

#### c.极低灰阶截断
四列 W/R/G/B，码值 0→30 渐变（SDR gamma 2.2）。检测极低灰阶是否发生截断。靠近顶部截断可基本忽略（实际上对使用没啥问题，感而且除非血轮眼应该没人看得清吧x），重点关注是否在靠近中心/底部就发生截断。

### 9. FPS Flicker
静止时保持低刷新率，长按触发高刷。目的：检测 LTPS 屏幕在刷新率改变时是否发生亮度跳变。
- 可选背景灰阶：G17 / G38 / G72 / G128 / G165
- 可隐藏帧率指示器（FPS 仅为 Choreographer 测量值，不严格准确）
主要是看刷新率改变时屏幕闪不闪，不过现在很多LTPS屏幕在低亮度、低APL下刷新率就不自动切了，得拉高一点点，最好在频闪切换点附近。
